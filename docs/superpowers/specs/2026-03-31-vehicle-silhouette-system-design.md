# Vehicle Silhouette System

## Summary

Replace the hand-coded generic SVG icons in `rolling-stock-svg.tsx` with ~46 detailed vehicle silhouettes extracted from Gemini-generated sprite sheets. Users select a specific silhouette when creating inventory items (locomotives, freight cars, etc.), and those silhouettes render throughout the app â€” inventory cards, train builder, consist views, and operating sessions.

## Extraction Pipeline

### Script: `scripts/extract-silhouettes.ts`

A Node.js script (run with `npx tsx`) that:

1. Parses each sprite sheet SVG using a DOM parser (e.g., `xmldom` or `cheerio`)
2. Computes bounding boxes for all `<path>` elements
3. Clusters spatially adjacent paths into groups (each group = one vehicle). Clustering approach: sort paths by centroid position, apply grid-based grouping since the sheets use a consistent row/column layout
4. Extracts each cluster as a standalone SVG with a tight viewBox and normalized coordinates
5. Processes both light and dark variants, outputting paired files
6. Names files using slugified vehicle names from the label text (detected from the sheet) or a manual mapping constant in the script

### Output

```
public/silhouettes/
  light/
    gp9.svg
    gp18.svg
    40ft-boxcar.svg
    ...
  dark/
    gp9.svg
    gp18.svg
    40ft-boxcar.svg
    ...
```

Each SVG is a self-contained silhouette with:
- Tight viewBox cropped to the vehicle bounds (no surrounding whitespace)
- Preserved aspect ratio
- Single `<g>` with all paths
- No fixed width/height attributes (scales to container)

## Schema Changes

### New Enum: `SilhouetteCategory`

```prisma
enum SilhouetteCategory {
  DIESEL_GP
  DIESEL_SD
  DIESEL_ERA
  STEAM
  FREIGHT_CAR
  PASSENGER_CAR
  CABOOSE
  MOW
}
```

### New Model: `Silhouette`

```prisma
model Silhouette {
  id          String              @id @default(cuid())
  name        String              // "GP38-2", "40ft Boxcar"
  slug        String              @unique // "gp38-2", "40ft-boxcar"
  category    SilhouetteCategory
  filePath    String              // "/silhouettes/light/gp38-2.svg"
  darkPath    String              // "/silhouettes/dark/gp38-2.svg"

  locomotives  Locomotive[]
  freightCars  FreightCar[]
  passengerCars PassengerCar[]
  cabooses     Caboose[]
  mowEquipment MOWEquipment[]

  createdAt   DateTime            @default(now())

  @@index([category])
}
```

### FK Additions

Add optional `silhouetteId` to each rolling stock model:

```prisma
// On Locomotive, FreightCar, PassengerCar, Caboose, MOWEquipment:
silhouetteId  String?
silhouette    Silhouette? @relation(fields: [silhouetteId], references: [id], onDelete: SetNull)
```

### Seed Script: `prisma/seed-silhouettes.ts`

Reads the `public/silhouettes/light/` directory, creates a `Silhouette` row for each file. Category is determined by a mapping constant (slug -> category). Run via `npx tsx prisma/seed-silhouettes.ts`.

## UI Components

### `SilhouettePicker`

Location: `components/ui/silhouette-picker.tsx`

A reusable grid component for selecting a silhouette:

- Fetches all silhouettes from a server action
- Displays as a responsive grid of thumbnail SVGs
- Optional category filter tabs (but all visible by default)
- Search/filter by name
- Click to select, shows selected state
- Renders light or dark variant based on current theme (via `next-themes` `useTheme()`)
- Props: `value?: string` (silhouetteId), `onChange: (id: string | null) => void`

### `SilhouetteImage`

Location: `components/ui/silhouette-image.tsx`

A display component that renders a silhouette by ID or slug:

- Accepts `silhouetteId` or `slug` prop
- Renders an `<img>` tag pointing to the correct light/dark file path
- Swaps src based on theme
- Fallback: if no silhouette assigned, renders nothing (parent component decides fallback behavior)
- Props: `silhouetteId?: string`, `slug?: string`, `className?: string`, `alt?: string`

### Integration Points

**Inventory Forms** (`locomotive-form.tsx`, `freight-car-form.tsx`, `caboose-form.tsx`, `passenger-car-form.tsx`, `mow-equipment-form.tsx`):
- Add `SilhouettePicker` to each form
- Store selected `silhouetteId` on the record

**Inventory Card Lists** (`locomotive-card-list.tsx`, `freight-car-card-list.tsx`, etc.):
- Replace generic icons with `SilhouetteImage` using the item's `silhouetteId`
- Fall back to current generic icon if no silhouette assigned

**Train Builder** (`components/consists/train-builder.tsx`):
- Replace `RollingStockIcon` usage with `SilhouetteImage`
- Fall back to current `RollingStockIcon` generic shapes if no silhouette assigned

**`RollingStockIcon`** (`components/consists/rolling-stock-svg.tsx`):
- Extend to accept optional `silhouetteId` prop
- When provided, render `SilhouetteImage` instead of the generic inline SVG
- Keeps backward compatibility â€” existing callers without silhouetteId still work

## Server Actions

### `getSilhouettes`

Location: `app/actions/silhouettes.ts`

```ts
export async function getSilhouettes(category?: SilhouetteCategory): Promise<Silhouette[]>
```

Returns all silhouettes, optionally filtered by category. No auth required (these are global built-in assets).

## Dark Mode Strategy

Each silhouette has paired light/dark SVG files. The `SilhouetteImage` component reads the current theme from `next-themes` and serves the appropriate variant. This avoids CSS filter hacks and ensures the dark mode rendering matches the original Gemini-generated assets exactly.

## What's Not In Scope

- User-uploaded custom silhouettes
- SVG colorization (silhouettes remain monochrome)
- Animated silhouettes
- Per-railroad paint scheme rendering
