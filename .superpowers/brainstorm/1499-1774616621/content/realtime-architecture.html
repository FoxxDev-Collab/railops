Real-Time Collaboration Architecture

Hybrid approach: Liveblocks for presence + broadcast, server actions for data

Room Architecture

Liveblocks Room Structure
railroad:{layoutId}
├── Presence: who's online, what page they're viewing
├── Broadcast: record-created, record-updated, record-deleted
├── Editing indicators: "Jeremy is editing Springfield Yard"
└── session:{sessionId}
├── Dispatcher: full train board, all assignments
├── Crew: "My Work" view, assigned trains only
└── Broadcast: car-moved, switch-list-completed, train-departed

Data Flow

How a record edit flows through the system
User A Opens "Edit Location: Springfield Yard"
↓ Presence update → Liveblocks → All clients see "User A is editing Springfield Yard"
User A Saves changes via server action
↓ Server action → Prisma write → DB updated (source of truth)
↓ Server broadcasts → Liveblocks → { type: "record-updated", section: "locations", id: "clx123" }
User B Receives broadcast → Invalidates cache → Refetches location data
↓ UI updates automatically with fresh data
