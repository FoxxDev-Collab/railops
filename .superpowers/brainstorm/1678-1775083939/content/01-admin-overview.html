Enterprise Admin Panel — System Overview

Here's the full scope of what we'll build. Which modules matter most? Select your top priorities.

1

Customer Lifecycle Management

Account states: Active, Suspended, Churned, Deactivated with automated transitions

Onboarding tracking: See where users drop off (signed up → verified → created layout → added stock → ran session)

Churn alerts: Flag users inactive 14+ days, subscription about to expire, downgrade signals

Account actions: Suspend/reactivate with reason tracking, grace periods, automated email notifications

New Prisma models: AccountStatus enum, AccountEvent log

2

Email & Communications Center

Template system: Welcome, verification, password reset, subscription changes, announcements — all editable

Compose & send: Write and send to individual users or segments (all free, all pro, unverified, inactive)

Broadcast announcements: New feature alerts, maintenance windows, newsletters to all users

Email log: Track every email sent — recipient, template, delivery status, opens

New Prisma models: EmailTemplate, EmailLog, EmailBroadcast

3

Analytics & Business Intelligence

Growth charts: Signups over time (daily/weekly/monthly), cumulative users, MRR trend line

Cohort analysis: Retention by signup month, conversion funnel (free → pro over time)

Feature usage: Most active features per user, layouts created, sessions run, rolling stock counts

Revenue dashboard: ARR projections, churn rate, LTV estimates, plan distribution trends

Computed from existing data — no new models needed, just new queries + chart components

4

Operational Health & Monitoring

Database dashboard: Table sizes, row counts, slow queries, connection pool status

System health: Uptime, memory usage, active sessions, API response times

Error tracking: Recent server errors, failed actions, rate limit hits

Background jobs: Email queue status, scheduled tasks, cleanup jobs

New Prisma models: SystemMetric (time-series), ErrorLog

5

Enhanced User Detail Page

Full profile view: Account timeline, all railroads with stats, login history, subscription history

Admin actions panel: Send email, adjust plan, add credit, extend trial, add notes

Usage breakdown: Resource counts by railroad, storage used, API calls, last active features

Support notes: Internal admin notes per user (not visible to user), tagged conversations

Extends existing user detail page + new AdminNote model

6

Feature Flags & Config

Toggle features: Enable/disable features per plan tier or globally without deploys

Beta access: Grant specific users access to unreleased features

Plan limits: Configure item limits (free: 25 items), storage caps, API rate limits from admin UI

A/B testing: Simple percentage-based rollout for new features

New Prisma models: FeatureFlag, UserFeatureOverride
