Charts & Data Visualization — Library Choice

Phase 1 needs charts for signup trends, MRR, cohort analysis, feature usage. Which charting approach?

A

Recharts

The most popular React charting library. Declarative, composable, built on D3. shadcn/ui already has chart components wrapping Recharts.

Already in shadcn/ui ~45kb gzipped Line, Bar, Area, Pie, Radar

Easiest integration path. Good enough for most SaaS dashboards. Customizable but can feel generic without effort.

B

Tremor

Dashboard-first React component library. Built specifically for analytics dashboards. Comes with KPI cards, sparklines, and data tables built in.

Dashboard-optimized Uses Recharts under the hood KPIs + Charts + Tables

Adds another component library alongside shadcn/ui. Great defaults but may conflict with your existing design system.

C

shadcn/ui Charts (Recharts wrapper)

Use shadcn/ui's built-in chart components. They wrap Recharts with your theme's CSS variables, ensuring charts match your design system automatically.

Zero new dependencies Matches existing theme Consistent with rest of UI

Best consistency with your existing shadcn/ui components. Same theming, same dark mode, same design language. Recommended.

My recommendation is C (shadcn/ui Charts) — zero new dependencies, automatic theme integration, and it keeps the admin panel visually consistent with the rest of RailOps. Adding Tremor alongside shadcn/ui creates two competing design systems.
